1. Importance of HTML5 Semantic Elements

10 pts

HTML5 semantic elements describe the meaning of content rather than just its appearance. Before HTML5, developers relied heavily on generic <div> and <span> tags with class names like class="header" or class="nav" to fake structure. Semantic tags such as <header>, <nav>, <section>, <article>, and <footer> replace that guesswork with elements the browser, screen readers, and search engines can actually understand.

  • Structure: Semantic tags create a predictable document outline. <header> typically holds a logo and introductory content, <nav> wraps primary navigation links, <section> groups thematically related content, <article> marks self-contained, independently distributable content (a blog post, a product card), and <footer> holds closing information such as copyright and contact details. A developer opening the file six months later can understand the layout at a glance instead of decoding a wall of nested <div>s.
  • Accessibility: Screen readers and other assistive technology use semantic elements to build a navigable outline of the page. A visually impaired user can jump straight to the <nav> landmark or skip past a repeated <header> to reach the <main> content, which is far harder to do when everything is an unlabeled <div>. This aligns with WCAG accessibility guidelines and makes the site usable by a wider audience.
  • SEO: Search engine crawlers weigh semantic structure when indexing a page. Content wrapped in <article> or <section> signals what is genuinely important versus decorative, and a properly used <nav> helps crawlers map site architecture. This can improve ranking and how a page's content is summarized in search results.

In short, semantic elements make a page easier to read for three audiences at once — developers, assistive technology, and search engines — which is why they are considered a baseline best practice in modern HTML5 development.

2. Responsive Web Design

10 pts

Responsive web design (RWD) is an approach to building websites so that layout, images, and content automatically adapt to the size and orientation of the device displaying them — a desktop monitor, tablet, or smartphone — without requiring separate versions of the site. The goal is one codebase that reflows gracefully across screen sizes rather than a fixed, one-size-fits-all layout.

Three CSS3 features make this practical:

  • Flexbox: A one-dimensional layout model ideal for aligning items in a row or column — navigation bars, button groups, or card rows that need to wrap neatly as space shrinks. Properties like flex-wrap and justify-content let elements resize and reflow automatically.
  • Grid Layout: A two-dimensional system for arranging content in rows and columns simultaneously, well suited to whole-page layouts such as a product gallery or dashboard. Using fr units and repeat(auto-fit, minmax(...)), a grid can go from four columns on a large screen to a single column on mobile with no media query at all.
  • Media Queries: Conditional CSS rules (@media (max-width: 768px) { ... }) that apply different styles based on the viewport's width, height, or orientation. They are what let a designer explicitly redefine font sizes, hide a sidebar, or stack a Flexbox row into a column at defined breakpoints.

Together, these tools let a single HTML document serve every device: Flexbox and Grid handle fluid arrangement, while media queries fine-tune the experience at specific breakpoints — the combination behind the "mobile-first" approach most modern sites follow today.

3. Planning an E-Commerce Website

10 pts

For a small business selling products online, I would structure the site around the following core pages, each serving a distinct step in the customer's journey:

  • Home Page — the first impression: hero banner, featured or best-selling products, brief brand story, and clear calls to action guiding visitors toward the Products page.
  • About Page — builds trust by explaining who the business is, its mission, and what makes it different, which matters for a small business competing against larger retailers.
  • Products Page — a catalog/grid of items with filtering and sorting (by category, price, popularity) so customers can browse efficiently.
  • Product Details Page — a dedicated page per item with photos, description, price, stock status, size/variant options, and an "Add to Cart" button.
  • Login / Register Page — account creation and sign-in so customers can save addresses, view order history, and check out faster on repeat visits.
  • Contact Page — a form plus business details (email, phone, social links, physical address/map) for customer support and inquiries.
  • Checkout Page — order summary, shipping information, and payment entry, designed to be short and distraction-free to reduce cart abandonment.

To make the site attractive and easy to use, I would apply these CSS3 features:

  • Color palette: a small, consistent set of brand colors (a primary, an accent, and a neutral background) applied through CSS custom properties so the whole site stays visually cohesive.
  • Typography: a readable body font paired with a distinct heading font, consistent sizing scale, and enough line-height for comfortable reading.
  • Layout: CSS Grid for the product catalog and Flexbox for navigation bars, filter controls, and the cart summary.
  • Hover and transition effects: subtle image zoom, button color shifts, or card elevation on hover to make the catalog feel interactive without being distracting.
  • Responsive design: media queries and fluid grids so browsing and checkout both work cleanly on a phone, since a large share of online shopping happens on mobile devices.

4. HTML vs. CSS and the Case for Separation

10 pts

HTML defines the meaning and hierarchy of content — headings, paragraphs, lists, images, forms — independent of how it appears visually. CSS is responsible for presentation: color, spacing, typography, layout, and responsive behavior. Keeping these concerns separate, rather than styling elements with inline style attributes scattered through the markup, is considered best practice for several concrete reasons:

  • Separation of concerns: A developer editing content in an HTML file does not need to touch design rules, and a designer adjusting colors or spacing does not need to edit markup — each file has one clear job.
  • Easier maintenance: A brand color change made once in a stylesheet (:root { --primary: #2f7a72; }) can update every button, heading, and link across an entire site instantly, instead of hunting through dozens of pages for inline styles.
  • Reusability: One external stylesheet (styles.css) can be linked from every page of a site, so a consistent look is achieved with a single <link rel="stylesheet" href="styles.css"> tag rather than rewriting styles per page.
  • Faster updates and performance: Browsers cache external CSS files, so pages load faster after the first visit, and teams can update the stylesheet without redeploying HTML content, and vice versa.

In practice, this is why professional projects use external .css files linked in the <head> rather than inline styles or heavy use of the <style> tag within individual pages.

5. Building User-Friendly, Accessible, Responsive Websites

10 pts

A website is only successful if real people can actually use it. A visually impressive site that is slow, confusing, or unusable on a phone will lose visitors regardless of how good the design looks on a desktop monitor. Prioritizing usability, accessibility, responsiveness, and visual appeal together — rather than treating any one of them as optional — directly affects how long visitors stay, whether they convert into customers, and whether the site reaches the widest possible audience, including users with disabilities and users on limited devices or connections.

Five best practices I would follow as a future web developer:

  • Consistent navigation: Keep the menu, logo placement, and layout patterns identical across every page so users always know where they are and how to get back.
  • Semantic HTML and accessibility considerations: Use proper heading order, alt text on images, labeled form fields, and sufficient color contrast so the site works with screen readers and keyboard navigation, not just a mouse.
  • Mobile-first, responsive layouts: Design for the smallest screen first, then progressively enhance for tablets and desktops using Flexbox, Grid, and media queries.
  • Readable typography and clear visual hierarchy: Choose legible font sizes and line spacing, and use size/weight/color to make headings, body text, and calls to action easy to tell apart at a glance.
  • Optimized images and organized, external CSS: Compress images and use modern formats to keep load times fast, and keep styles in external, well-organized stylesheets so the codebase stays maintainable as the site grows.

Together these practices ensure a site is not just attractive, but genuinely usable — which is the actual measure of good web design.
